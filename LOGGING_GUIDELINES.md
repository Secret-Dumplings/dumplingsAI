# 日志级别使用规范

本文档规范 dumplingsAI 项目中日志级别的使用，避免日志滥用导致的信息过载或敏感信息泄露。

## 日志级别定义

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| **TRACE** | 最详细的调试信息，默认关闭 | 流式响应每个 chunk、完整响应 JSON、装饰器内部流程 |
| **DEBUG** | 开发调试信息，不包含敏感数据 | 对象长度、工具调用参数（脱敏）、流程状态 |
| **INFO** | 关键业务节点、状态变更 | Agent 连接状态、服务启动/停止 |
| **WARNING** | 潜在问题但不影响运行 | 工具未找到、配置异常 |
| **ERROR** | 错误但程序可继续 | 单个工具执行失败、API 请求错误 |
| **CRITICAL** | 严重错误需立即处理 | 核心服务崩溃、数据丢失 |

## 禁止行为

1. **禁止使用 INFO 级别记录以下内容：**
   - 完整的 prompt 内容
   - 完整的对话历史 (history)
   - 工具调用的完整参数（可能含敏感信息）
   - API 响应的完整内容

2. **禁止使用 DEBUG 级别记录以下内容：**
   - 流式响应的每一行数据（应使用 TRACE）
   - 完整的 JSON 响应体（应使用 TRACE）

## 配置日志级别

### 通过环境变量

```bash
# 开发环境：显示 DEBUG 及以上
export LOGURU_LEVEL=DEBUG

# 生产环境：显示 INFO 及以上
export LOGURU_LEVEL=INFO

# 详细调试：显示 TRACE 及以上
export LOGURU_LEVEL=TRACE
```

### 通过代码

```python
from dumplingsAI.logging_config import setup_logging

# 开发环境
logger = setup_logging(level="DEBUG")

# 生产环境
logger = setup_logging(level="INFO")
```

## 日志脱敏

记录工具调用参数或其他可能包含敏感信息的数据时，应进行脱敏处理：

```python
# 错误示例
logger.info(f"调用工具，参数：{api_key}")

# 正确示例
logger.debug(f"调用工具，参数：{sanitize_data(args)}")
```

## 日志格式

统一使用 `logging_config.py` 中定义的格式：

- **控制台格式：** 时间 | 级别 | 模块：函数：行号 - 消息
- **文件格式：** 包含完整堆栈跟踪和诊断信息
